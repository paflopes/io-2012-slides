Não tínhamos todo o conhecimento necessário para criar um aplicativo móvel e nem o tempo necessário para que a equipe aprendesse. O tempo não estava a nosso favor.

A equipe

  • Sem conhecimento prévio da plataforma Android, iOS ou WP
  • Sem tempo para aprender C# e Objective-C
  • Conhecimento em Python
  • Linguagens utilizadas no projeto: Java, JavaScript, HTML

Ionic é um SDK para construção de aplicativos híbridos móveis com HTML5.

A solução

Quem usa?

Ionic é um SDK opensource para construção de aplicativos híbridos móveis com HTML5. Tem uma performance muito boa, zero JQuery, apesar de ser compatível.

Ele utiliza as tecnologias

  • AngularJS: para criação de aplicações robustas. Oferece injeção de dependências, modularização, data binding, etc.
  • NgCordova: módulo do AngularJS para acesso de recursos nativos da plataforma. Existem recursos para TouthID da Apple, Camera, OAuth e mais.
  • CLI: Interface de linha de comando para tarefas comuns. Live reload, mesmo rodando nos dispositivos móveis, scafolding.

Ionic

  • O spring framework já era utilizado no código anterior.
  • Groovy foi escolhido como linguagem para a api por ser mais legível e totalmente compatível com Java. Assim foi possível reutilizar o código já existente.
  • Gulp já vem com os projetos ionic. Utilizamos principalmente para compilar o CoffeeScript.
  • Foi escolhido pela sua facilidade de programação, e para evitar alguns erros típicos que teríamos com Javascript puro.
